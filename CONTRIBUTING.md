# Contributing

This tap is **managed by [GoReleaser](https://goreleaser.com/)** from the individual
tool repositories. Please do not edit files under `Formula/` or `Casks/` directly —
any manual changes will be overwritten by the next release.

## How releases land here

Each tool repo (e.g. `tfpluginschema`, `tfmoduleschema`) has a GoReleaser
configuration with a `brews:` block pointing at this tap:

```yaml
brews:
  - repository:
      owner: matt-FFFFFF
      name: homebrew-tap
      branch: main
      token: "{{ .Env.HOMEBREW_TAP_TOKEN }}"
    directory: Formula
    homepage: https://github.com/matt-FFFFFF/<tool>
    description: <short description>
    license: <license>
```

On release, GoReleaser authenticates as a GitHub App installed on this repo
(with `contents: write`) and commits the updated formula to `Formula/`.

## README auto-update

The `## Formulae` section of [`README.md`](./README.md) is regenerated by
[`.github/workflows/update-readme.yml`](./.github/workflows/update-readme.yml)
whenever files under `Formula/**` or `Casks/**` change on `main`. The workflow
runs [`scripts/update-readme.sh`](./scripts/update-readme.sh), which:

1. Parses `desc`, `homepage`, and `version` from each `*.rb` file.
2. Rewrites the content between the `<!-- FORMULAE:START -->` and
   `<!-- FORMULAE:END -->` markers.
3. Commits the result using `GITHUB_TOKEN` (which, by design, does not trigger
   further workflow runs — avoiding a loop).

Because GoReleaser's commit is made with a GitHub App token, it _does_ trigger
the README workflow; the subsequent README commit with `GITHUB_TOKEN` does not.

## Making changes to the tap itself

Changes to the README template, workflow, or script are welcome via PR.
Anything inside `Formula/` or `Casks/` should be changed upstream in the
tool's repo and re-released.
